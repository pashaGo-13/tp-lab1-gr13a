14. - Команды перенаправления:
      - ``` > ``` - перезаписывает в файл 
      - ``` >> ``` - добавить в конец файла
      - ``` 2> ``` - перенаправить поток ошибок
      - ``` &> ``` - перенаправить и поток вывода, и поток ошибок
      - ``` < ``` - прочитать данные из файла
    - Команды группировки:
      - ``` | ``` - передает выводн одной команды на вход другой
      - ``` ; ``` - последовательно выполнить команды
      - ``` && ``` - выполнить вторую команду, только если первая выполнена успешно
      - ``` || ``` - как предыдущая, но в случае неуспеха

16. Когда нужно создать проект под управлением гит или управлять версиями уже созданного. Команда - ``` git init ```
17. Коммит - это операция сохранения текущего состояния проекта в историю Git.
18. В Git ветка является лишь указателем на определенный коммит, поэтому не сделав коммит, измнения останутся в рабочей директории и при переходе на другую ветку эти изменения могут перейти туда же, создав конфликт.
19. Состояния и переходы к следующим:
      - Изменен (modified) - изменен, но не добавлен в индекс - ``` git add <имя> ```
      - Индексирован (staged) - добавлен в индекс - ``` git commit ```
      - Закоммичен (commited) - изменения сохранены в репозитории. 
20. Ветка - это подвижный указатель на один из коммитов. С каждым новымм коммитом указатель автоматически перемещается вперед. Ветки позволют вести одну работу параллельно.
21. Обязательно выполнение одного коммита
22. HEAD - это специальный указатель, ссылающийся на текущий коммит (соответственно, ветку).
23. - Создать, но не переключаться - ``` git branch <имя> ```
    - Создать и переключить - ``` git checkout -b <имя> ```
    - Создать от определенного коммита - ``` git branch <имя> <хеш коммита> ```
24. ``` git branch ``` - показывает все ветки, текущая будет отмечена другим цветом и *.
25. ``` git checkout ```
26. ``` git merge <имя которую хотим влить в ту, в которой находимся> ```
27. ``` git remote add origin <SSH репозитория> ``` - origin это имя репозитория
28. - ``` git reset HEAD~1 ``` - отменить коммит, изменения оставить в рабочей директории
    - ``` git reset --soft HEAD~1 ``` - отменить коммит, изменения оставить в индексе
    - ``` git reset --hard HEAD~1 ``` - удалить последний комммит и все изменения безвозвратно
    - ``` git revert HEAD ``` - создать новый коммит, отменяющий изменения последного
29. Для перебазирования нескольких коммитов на новый базовый коммит (нужен, чтобы не засорять историю коммитов мерж-коммитом)
30. Форк нужен, когда нужно изменить чужой проект, но доступ к изменению в оригинальном репозитории закрыт. Создается копия (форк), а затем после изменений предлагается внесение изменений в оргинальном репозитории (пул реквест).
Способы создания:
  - через сайт GitHub
  - создать форк на сайте и клонировать с помощью ``` git clone ```
  - с помощью GitHub CLI ``` gh repo fork оригинальный_владелец/репозиторий ```
